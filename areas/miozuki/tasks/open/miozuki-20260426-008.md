One-time Claude Code setup sprint for the miozuki-web repo. Groups all developer tooling tasks.

**1. Shopify Dev MCP (5 minutes)**
```bash
claude mcp add --transport stdio shopify-dev-mcp -- npx -y @shopify/dev-mcp@latest
```
Verify with `claude mcp list`. Gives Claude Code access to Shopify Admin API, GraphQL schema, and Storefront API docs. No manual copy-pasting of product specs.

**2. Vercel Plugin for Claude Code (10 minutes)**
Check vercel.com/docs/integrations/claude-code for current install method.
Adds: /deploy, /env, /status, /marketplace commands inside Claude Code sessions.

**3. miozuki-web CLAUDE.md (1-2 hours)**
Create or update CLAUDE.md in the miozuki-web repo root with:
- Shopify Storefront API version in use
- Product metafield schema
- Vercel project name and environment names (production, preview, development)
- NZ English: jewellery, colour, organise, recognise, etc.
- Brand voice: editorial, warm but not fluffy, ethical sourcing angle. Never: "stunning", "gorgeous", "perfect", "unique"
- Canonical pattern for collection and product pages
- Subdirectory architecture rule: content hubs go at /learn or /moissanite-guide, never as subdomains
- Internal linking convention

**4. Hooks in .claude/settings.json (1 hour)**

Auto-commit hook -- PostToolUse on Write:
Automatically commits every file written by Claude Code. Audit trail for AI-generated content.

Vale prose linter hook -- PostToolUse on Write:
Runs Vale on every markdown file written. Catches NZ English violations before content reaches review.
Prereq: install Vale (`scoop install vale` on Windows).

PreToolUse gate:
Blocks git push and Shopify deployment inside Claude Code sessions until a condition is met. Safety rail for the production repo.

**5. Custom slash commands in .claude/commands/ (2-3 hours)**

- `/product-description [product]`: brand-voice product copy, 150-200 words, NZ English, ethical sourcing angle
- `/seo-meta [page]`: title tag, meta description, OG tags
- `/blog-outline [topic]`: structured moissanite/pearl education article outline with H2/H3 and FAQ section
- `/faq-entry [question]`: FAQ schema-ready Q&A pair in JSON-LD FAQPage format

Note: use `.claude/skills/<name>/SKILL.md` format (current Claude Code convention) rather than the legacy `.claude/commands/` format.
