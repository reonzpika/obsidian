# Medtech / ALEX Glossary

Reference for ClinicPro Medtech development. Terms specific to the NZ health ecosystem and the ALEX API integration.

---

| Term | Definition |
|------|-----------|
| **ALEX** | Medtech's API platform — the vendor integration layer used by all ClinicPro Medtech tools |
| **BFF** | Backend-for-frontend — AWS Lightsail server at api.clinicpro.co.nz (static IP, allow-listed by Medtech). All ALEX API calls must route through the BFF |
| **FHIR** | Clinical data standard used by the ALEX API |
| **NHI** | National Health Index — NZ patient identifier (e.g. ZZZ0016 for test patient) |
| **HPI** | Health Practitioner/Provider Index — NZ facility and practitioner identifier system |
| **facilityId** | HPI facility identifier passed in every ALEX request header (e.g. F99669-C, F2N060-E) |
| **F99669-C** | Local Medtech Evolution facility — use for UI validation (Inbox, Daily Record visible) |
| **F2N060-E** | Medtech hosted UAT facility — use for API-only checks; not visible in local Evolution UI |
| **DOM referral** | Legacy referral system in Medtech Evolution / Medtech32 |
| **Inbox Scan** | ALEX write-back target — a POST DocumentReference creates a document in the patient's Evolution Inbox Scan folder (appears inline) |
| **MT Icon Loader** | Medtech tool to load custom toolbar icons into Evolution |
| **ALEX Apps Config** | ALEX configuration panel for registering vendor app launch URLs |
| **Hybrid Connection** | Azure Hybrid Connection Manager — tunnels to local F99669-C Evolution instance; must be running on Ryo's desktop for local testing |
| **correlationId** | Request tracing ID generated by the Vercel commit route, forwarded to ALEX as the `mt-correlationid` header |
| **media** | FHIR Media resource — used for image write; appears as a View link only in Evolution (not inline). Do not use for Inbox Scan writes |
| **DocumentReference** | FHIR DocumentReference resource — used for Inbox Scan write-back (TIFF/PDF format); appears inline in Evolution. This is the correct write target |
| **ClinicPro Capture** | The active Medtech product — standalone mobile web app for clinical photo capture and upload to Medtech Evolution via FHIR ALEX API. App route: `app/(medtech-standalone-mobile-image-webbapp)/`. Note: `app/(medtech)/` is legacy, do not use |
