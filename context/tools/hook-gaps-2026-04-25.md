## Top 5 hooks to implement

### 1. Git push gate (PreToolUse, Impact x Ease = 20)

Fires before any Bash call matching `git push`. Outputs an error message with exit code 2, which causes Claude Code to block the tool call and show the message to the user. The user must explicitly type "yes" in their next message.

**Assumption (90% confidence):** PreToolUse with exit code 2 blocks the tool and surfaces stdout to the user. If this is exit code 1, the blocking behaviour is the same but the message surface differs. Verify against official hooks docs before deploying.

```python
#!/usr/bin/env python3
# C:/Users/reonz/Cursor/hooks/hook_git_push_gate.py
"""
PreToolUse hook: blocks git push, git push --force, and git push -f
unless the user's prior message contains an explicit 'yes' confirmation.

Exit codes (Claude Code hook protocol):
  0 = allow
  2 = block and show stdout as error to user
"""

import json
import sys
import re


BLOCKED_PATTERNS = [
    r"\bgit\s+push\b",
    r"\bgit\s+.*--force\b",
    r"\bgit\s+.*-f\b",
]

CONFIRMATION_PHRASES = [
    "yes, push",
    "confirm push",
    "go ahead and push",
    "yes push",
    "push it",
]


def main():
    try:
        payload = json.load(sys.stdin)
    except Exception:
        sys.exit(0)

    if payload.get("tool_name") != "Bash":
        sys.exit(0)

    command = payload.get("tool_input", {}).get("command", "")

    is_push = any(re.search(p, command, re.IGNORECASE) for p in BLOCKED_PATTERNS)
    if not is_push:
        sys.exit(0)

    # Check the most recent user message for explicit confirmation
    messages = payload.get("messages", [])
    last_user_text = ""
    for msg in reversed(messages):
        if msg.get("role") == "user":
            content = msg.get("content", "")
            if isinstance(content, list):
                for block in content:
                    if isinstance(block, dict) and block.get("type") == "text":
                        last_user_text = block.get("text", "").lower()
                        break
            elif isinstance(content, str):
                last_user_text = content.lower()
            break

    confirmed = any(phrase in last_user_text for phrase in CONFIRMATION_PHRASES)
    if confirmed:
        sys.exit(0)

    print(
        "BLOCKED: git push requires explicit confirmation.\n"
        "This is an irreversible action per CLAUDE.md rules.\n"
        "Confirm by including 'yes, push' in your next message."
    )
    sys.exit(2)


if __name__ == "__main__":
    main()
```

### 2. Destructive bash gate (PreToolUse, Impact x Ease = 20)

Blocks `rm -rf`, `Remove-Item -Recurse -Force`, and bulk delete patterns. Same exit-code-2 blocking protocol.

```python
#!/usr/bin/env python3
# C:/Users/reonz/Cursor/hooks/hook_destructive_bash_gate.py
"""
PreToolUse hook: blocks destructive shell commands without explicit confirmation.

Patterns blocked:
  - rm -rf / rm -fr
  - Remove-Item -Recurse or -Force
  - git reset --hard
  - git commit --amend
  - rmdir /s (Windows)
"""

import json
import sys
import re


BLOCKED_PATTERNS = [
    r"\brm\s+.*-[a-zA-Z]*r[a-zA-Z]*f\b",     # rm -rf, rm -fr, rm -Rf
    r"\brm\s+.*-[a-zA-Z]*f[a-zA-Z]*r\b",
    r"\bRemove-Item\b.*-Recurse\b",
    r"\bRemove-Item\b.*-Force\b",
    r"\bgit\s+reset\s+--hard\b",
    r"\bgit\s+commit\s+.*--amend\b",
    r"\brmdir\s+/[sS]\b",
]

CONFIRMATION_PHRASES = [
    "yes, delete",
    "yes delete",
    "confirm delete",
    "yes, reset",
    "confirm reset",
    "yes, amend",
    "confirm amend",
    "yes, proceed",
    "go ahead",
]


def main():
    try:
        payload = json.load(sys.stdin)
    except Exception:
        sys.exit(0)

    if payload.get("tool_name") != "Bash":
        sys.exit(0)

    command = payload.get("tool_input", {}).get("command", "")

    matched = next(
        (p for p in BLOCKED_PATTERNS if re.search(p, command, re.IGNORECASE)),
        None
    )
    if not matched:
        sys.exit(0)

    messages = payload.get("messages", [])
    last_user_text = ""
    for msg in reversed(messages):
        if msg.get("role") == "user":
            content = msg.get("content", "")
            if isinstance(content, list):
                for block in content:
                    if isinstance(block, dict) and block.get("type") == "text":
                        last_user_text = block.get("text", "").lower()
                        break
            elif isinstance(content, str):
                last_user_text = content.lower()
            break

    confirmed = any(phrase in last_user_text for phrase in CONFIRMATION_PHRASES)
    if confirmed:
        sys.exit(0)

    print(
        f"BLOCKED: destructive command detected:\n  {command[:120]}\n"
        "This is an irreversible action per CLAUDE.md rules.\n"
        "Confirm by including a confirmation phrase (e.g. 'yes, delete') in your next message."
    )
    sys.exit(2)


if __name__ == "__main__":
    main()
```

### 3. Em dash detector (PostToolUse, Impact x Ease = 20)

Fires after every Write or Edit tool call. Scans the file content written (or the new_string in an Edit) for em dashes. Prints a warning to stdout (Claude Code injects this as context). Uses exit code 0 (non-blocking) to avoid disrupting flow, but the warning is visible.

```python
#!/usr/bin/env python3
# C:/Users/reonz/Cursor/hooks/hook_em_dash_detector.py
"""
PostToolUse hook: scans written/edited content for em dashes (U+2014).
Non-blocking (exit 0) but prints a visible warning.

Covers:
  - Write tool: reads tool_input.content
  - Edit tool: reads tool_input.new_string
"""

import json
import sys
import re

EM_DASH = "—"
EN_DASH = "–"


def main():
    try:
        payload = json.load(sys.stdin)
    except Exception:
        sys.exit(0)

    tool_name = payload.get("tool_name", "")
    tool_input = payload.get("tool_input", {})

    if tool_name == "Write":
        content = tool_input.get("content", "")
        file_path = tool_input.get("file_path", "")
    elif tool_name == "Edit":
        content = tool_input.get("new_string", "")
        file_path = tool_input.get("file_path", "")
    else:
        sys.exit(0)

    found_em = content.count(EM_DASH)
    found_en = content.count(EN_DASH)

    if found_em == 0 and found_en == 0:
        sys.exit(0)

    parts = []
    if found_em:
        parts.append(f"{found_em} em dash(es) (—)")
    if found_en:
        parts.append(f"{found_en} en dash(es) (–)")

    # Show the first offending line for context
    lines = content.split("\n")
    first_hit = next(
        (ln.strip() for ln in lines if EM_DASH in ln or EN_DASH in ln), ""
    )

    print(
        f"WARNING: {', '.join(parts)} found in {file_path}.\n"
        f"  First hit: {first_hit[:100]}\n"
        "CLAUDE.md rule: no em dashes. Use commas, colons, or restructure."
    )
    sys.exit(0)


if __name__ == "__main__":
    main()
```

### 4. Bash command audit log (PreToolUse, Impact x Ease = 20)

Logs every Bash tool call to a rotating daily log file. Non-blocking (exit 0). Useful for reviewing what ran in a session and debugging unexpected side effects.

```python
#!/usr/bin/env python3
# C:/Users/reonz/Cursor/hooks/hook_bash_audit_log.py
"""
PreToolUse hook: writes every Bash command to a daily audit log.
Non-blocking (exit 0). Log location: ~/.claude/logs/bash-audit-YYYY-MM-DD.log
"""

import json
import sys
from pathlib import Path
from datetime import datetime, timezone


LOG_DIR = Path.home() / ".claude" / "logs"


def main():
    try:
        payload = json.load(sys.stdin)
    except Exception:
        sys.exit(0)

    if payload.get("tool_name") != "Bash":
        sys.exit(0)

    command = payload.get("tool_input", {}).get("command", "")
    session_id = payload.get("session_id", "unknown")
    cwd = payload.get("cwd", "")
    now = datetime.now(timezone.utc)

    LOG_DIR.mkdir(parents=True, exist_ok=True)
    log_file = LOG_DIR / f"bash-audit-{now.strftime('%Y-%m-%d')}.log"

    entry = (
        f"[{now.isoformat()}] session={session_id[:8]} "
        f"cwd={cwd}\n  CMD: {command[:500]}\n"
    )

    try:
        with log_file.open("a", encoding="utf-8") as f:
            f.write(entry)
    except Exception:
        pass

    sys.exit(0)


if __name__ == "__main__":
    main()
```

### 5. Session stats on Stop (Stop event, Impact x Ease = 15)

Fires on Stop. Reads the transcript and prints a summary: tool calls by type, files edited, session cost, duration. Output appears as a final context message.

```python
#!/usr/bin/env python3
# C:/Users/reonz/Cursor/hooks/hook_session_stats.py
"""
Stop hook: prints session stats to stdout so Claude Code surfaces them
as end-of-session context.

Stats: duration, total tool calls, edit/write count, bash count,
unique files touched, session cost (if available in payload).
"""

import json
import sys
from pathlib import Path
from datetime import datetime
from collections import Counter


def main():
    try:
        payload = json.load(sys.stdin)
    except Exception:
        sys.exit(0)

    transcript_path = payload.get("transcript_path", "")
    if not transcript_path:
        sys.exit(0)

    transcript = Path(transcript_path)
    if not transcript.exists():
        sys.exit(0)

    tool_counts = Counter()
    files_touched = set()
    first_ts = last_ts = None
    cost_usd = payload.get("cost_usd")

    try:
        with transcript.open(encoding="utf-8", errors="replace") as f:
            for line in f:
                try:
                    msg = json.loads(line)
                except Exception:
                    continue
                ts = msg.get("timestamp")
                if ts:
                    if first_ts is None:
                        first_ts = ts
                    last_ts = ts
                content = msg.get("message", {}).get("content", [])
                if not isinstance(content, list):
                    continue
                for block in content:
                    if not isinstance(block, dict):
                        continue
                    if block.get("type") != "tool_use":
                        continue
                    name = block.get("name", "unknown")
                    tool_counts[name] += 1
                    fp = block.get("input", {}).get("file_path", "")
                    if fp:
                        files_touched.add(fp)
    except Exception:
        sys.exit(0)

    if not tool_counts:
        sys.exit(0)

    duration_str = "unknown"
    if first_ts and last_ts:
        try:
            first = datetime.fromisoformat(first_ts.replace("Z", "+00:00"))
            last = datetime.fromisoformat(last_ts.replace("Z", "+00:00"))
            mins = int((last - first).total_seconds() // 60)
            duration_str = f"{mins} min"
        except Exception:
            pass

    edit_count = sum(
        tool_counts.get(t, 0) for t in ("Edit", "Write", "MultiEdit", "NotebookEdit")
    )
    bash_count = tool_counts.get("Bash", 0)
    total = sum(tool_counts.values())

    lines = [
        "--- Session stats ---",
        f"Duration: {duration_str}",
        f"Total tool calls: {total}",
        f"  Edits/Writes: {edit_count}",
        f"  Bash: {bash_count}",
        f"  Files touched: {len(files_touched)}",
    ]
    if cost_usd is not None:
        lines.append(f"  Cost: USD {cost_usd:.4f}")
    top = tool_counts.most_common(5)
    lines.append("Top tools: " + ", ".join(f"{k}={v}" for k, v in top))

    print("\n".join(lines))
    sys.exit(0)


if __name__ == "__main__":
    main()
```

## Notes and caveats

- All skeleton scripts above are **first drafts for human review**. Test each in a throwaway session before relying on them.
- The PreToolUse blocking mechanism (exit code 2) is assumed from the official hooks protocol. Verify against current docs before deploying gates 1 and 2.
- The `messages` array on the hook payload (used for confirmation detection in gates 1 and 2) may not be available in all Claude Code versions. If absent, the gate will always block, which is the safe failure mode.
- Hook scripts should be placed in a dedicated directory (`C:/Users/reonz/Cursor/hooks/`) separate from skill-evolver. Create the directory before adding settings.json entries.
- The `cost_usd` field in the Stop payload may not exist in all versions. The session stats script handles this gracefully.
- Frontmatter schema validator (gap 7) is excluded from top 5 because it requires maintaining a schema definition per file type, which adds ongoing maintenance cost. Recommend revisiting once the other gates are stable.

All output is a first draft for human review.
