Improve skills architecture based on skills-auto-improve and business-breakdown reports.

Three layers:

1. Activation audit: review YAML trigger descriptions on all skills. Community testing found activation as low as 20% with vague descriptions. Skill-creator skill has a built-in loop to improve trigger descriptions.

2. Evals: create evals/ folder inside each skill with evals.json containing binary assertions. Binary (not subjective) assertions only: "under 300 words", "does not contain em dashes", "final line is not a question".

3. Self-improvement loop: adapt Karpathy AutoResearch pattern. Claude modifies SKILL.md, runs evals, keeps if score improves, reverts if not. Can run overnight.

Start with highest-frequency skills (daily, weekly, session-update, obsidian).
