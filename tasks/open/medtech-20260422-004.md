Four env vars needed in Vercel for the duplicate NHI alert email feature to work. Without these, the server-side Resend alert fires silently (log-and-continue) and the picker flag is off.

Set in Vercel dashboard under clinicpro-medtech project settings, Production environment:

1. `RESEND_API_KEY` — Resend API key (already in `.env` locally)
2. `ALERT_EMAIL_TO_MEDTECH` — Medtech escalation recipient for duplicate NHI alerts (e.g. alexsupport@medtechglobal.com or Defne's address)
3. `ALERT_EMAIL_TO_INTERNAL` — ClinicPro internal recipient (e.g. ryo@clinicpro.co.nz)
4. `NEXT_PUBLIC_ENABLE_NHI_PICKER` — set to `true` to show the duplicate NHI picker modal on /search (currently feature-flagged off)

Deploy after setting.
