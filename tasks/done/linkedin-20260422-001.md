Build the fully autonomous posting pipeline so execution happens without the user at the laptop.

## Architecture agreed

Task Scheduler fires `scripts/execute_scheduled.py` at T-40min before each post slot.

Script phases:
1. Run `scripts/scout.py` to get fresh targets
2. Call `claude -p` to draft 6 GH comments (same subprocess pattern as grow workflow)
3. Send comments to Telegram for approval
4. Poll Telegram for reply — 15 min timeout, post as-is on no reply
5. On "approve" or timeout: run `scripts/assemble_session_state.py` then `execute_post.py`
6. On edit instruction: apply edit, resend to Telegram, restart poll loop

## Build order

1. Set up Telegram bot (`tools/telegram.py` — send + poll utility)
2. Write `scripts/execute_scheduled.py` orchestrating the above phases
3. Register Task Scheduler task at T-40min per session (replace current Claude Desktop wake task)
4. Smoke test end-to-end with a real session in dry-run mode

## Design decisions still needed

Before coding, plan session to agree on:
- Exact timeout duration (proposal: 15 min)
- Edit message format (proposal: "3: [new comment text]" or natural language)
- Fallback behaviour if scout returns 0 targets
- Fallback behaviour if `claude -p` GH draft fails
- Whether to replace or sit alongside current scheduler setup
