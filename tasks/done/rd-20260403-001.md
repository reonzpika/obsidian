Define the Inbox Helper task in precise, testable terms. This is Step 1 of the Objective 1 roadmap and the foundation for all architecture evaluation and synthetic data work.

## What to define

**Input document types**
- Enumerate every inbox item type the system will process (e.g. lab results, discharge summaries, outpatient letters, radiology reports, ACC correspondence, patient messages, referral acknowledgements, medication reviews)
- Explicitly list exclusions — what item types are out of scope for MVP

**Urgency taxonomy**
- Define number of urgency levels and their labels (e.g. Immediate / Urgent / Routine / Non-clinical)
- Write a clinical definition for each level: what makes an item "urgent" vs "routine"?
- Specify boundary cases: e.g. abnormal result that is expected/known vs unexpected; normal result but action still needed

**Output schema**
- Define exactly what the AI returns for each item: urgency label + any structured action recommendation
- Specify format (plain text, structured JSON, etc.)

**Success criteria — operationalise the target**
- Choose the metric: weighted F1? Sensitivity for Immediate/Urgent items? Cohen's kappa? Justify the choice.
- Define the test set composition: how many items, what distribution across urgency levels?
- Define what a "correct" prediction means: exact match only, or partial credit for adjacent-level errors?

**Deliverable:** A written specification document. This serves as MBIE evidence for Step 1 of Objective 1.
